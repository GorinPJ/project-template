### 質問3: アーキテクチャ

**選択肢**:
1. SPA + API分離（React/Vue + バックエンドAPI）
2. SSR/フルスタックJS（Next.js / Nuxt）
3. 従来型Webアプリ（PHP / Java / Ruby / Python）
4. 静的サイト + API
5. Shopifyアプリ（Remix/Next.js）
6. その他

**議論**:
- ユーザー: 「Shopifyアプリとは何？」
- 説明: Shopify Admin内に埋め込むアプリはNode.js系（Remix/Next.js）が前提
- ユーザー: 「Shopify Admin内に埋め込まない方法はある？」
- 説明: スタンドアロン方式なら独立したWebアプリとしてPHPでも構築可能
- ユーザー: 「Shopify Admin内に埋め込むことは簡単？また、独自の管理機能はPHPで実装したいが、実現できる？」
- 説明: 埋め込みはNode.js必須。PHPならスタンドアロン方式で実現可能
- ユーザー: 「スタンドアロン方式にしようかな。その上で、フロントにはVueJSを使ってみたい。バックグラウンドはPHP」

**回答**: `1`（SPA + API分離）

**決定した構成**:
- フロントエンド: Vue.js
- バックエンド: PHP
- Shopify連携: API経由（埋め込みではない）

## 設計判断のポイント

1. **Shopifyアプリ方式の選択**: 埋め込み型ではなくスタンドアロン方式を採用。PHPでの実装自由度を優先。

2. **フロントエンド技術**: Vue.js 3を採用。学習目的も含む。

3. **本番環境の考慮**: Windows Serverでの運用を前提に、cronではなくタスクスケジューラを使用する設計。

4. **段階的なCI/CD導入**: Phase 1でCI（品質チェック）のみ、Phase 2で自動デプロイを追加。
