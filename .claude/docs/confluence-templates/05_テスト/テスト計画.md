# テスト計画

## 概要

本プロジェクトのテスト計画を記載します。

---

## テスト方針

### 目的

- 品質の確保と不具合の早期発見
- リグレッションの防止
- ユーザー要件の充足確認

### 基本方針

- テストピラミッドに基づく効率的なテスト設計
- 自動テストを最大限活用
- 重要度の高い機能から優先的にテスト

---

## テスト種別

### テストレベル

| レベル | 目的 | 担当 | 自動化 |
|-------|------|------|--------|
| 単体テスト | 関数・クラスの動作確認 | 開発者 | ○ |
| 結合テスト | コンポーネント間連携確認 | 開発者 | ○ |
| システムテスト | システム全体の動作確認 | QA | △ |
| 受入テスト (UAT) | 要件充足確認 | ユーザー | × |

### テスト種別

| 種別 | 内容 | タイミング |
|------|------|-----------|
| 機能テスト | 機能要件の確認 | 各フェーズ |
| 性能テスト | レスポンス、スループット確認 | リリース前 |
| セキュリティテスト | 脆弱性診断 | リリース前 |
| 回帰テスト | 既存機能への影響確認 | 変更時 |

---

## テスト観点

### 機能テスト観点

| 観点 | 内容 |
|------|------|
| 正常系 | 期待通りの入力で期待通りの出力 |
| 異常系 | 不正入力時のエラーハンドリング |
| 境界値 | 上限・下限値での動作 |
| 同値分割 | 代表値での動作確認 |

### 非機能テスト観点

| 観点 | 確認内容 |
|------|---------|
| 性能 | レスポンスタイム、同時接続数 |
| セキュリティ | SQLインジェクション、XSS、認証・認可 |
| ユーザビリティ | 操作性、エラーメッセージの分かりやすさ |
| 互換性 | ブラウザ、デバイス対応 |

## 不具合管理

### 重要度定義

| 重要度 | 定義 | 対応期限 |
|-------|------|---------|
| Critical | システム停止、データ損失 | 即時 |
| Major | 主要機能が使用不可 | 24時間以内 |
| Minor | 機能の一部に問題あり | リリースまで |
| Trivial | 軽微な問題（誤字など） | 次回リリース |

### 不具合起票ルール

| 項目 | 内容 |
|------|------|
| タイトル | [画面名] 不具合の概要 |
| 再現手順 | 1. xxx 2. xxx 3. xxx |
| 期待結果 | 期待される動作 |
| 実際の結果 | 実際に起きた動作 |
| 環境 | ブラウザ、OS、環境 |
| 証跡 | スクリーンショット、ログ |
