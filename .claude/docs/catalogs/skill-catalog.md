# スキルカタログ

利用可能なスキル一覧です。スキルはエージェントが自動的に参照するルール・ガイドラインです。

## スキルとエージェントの違い

| 項目 | スキル | エージェント |
|------|--------|------------|
| 呼び出し方 | 自動適用 | 明示的に呼び出し |
| 用途 | ルール・ガイドライン | タスク実行 |
| 例 | コーディング規約 | 機能実装 |

## スキル一覧

### coding-standards（コーディング規約）

コード作成時に自動適用される命名規則・スタイルガイド。

**適用タイミング**: コード生成・編集時

**主な内容**:
- 命名規則（変数、関数、クラス）
- インデント・フォーマット
- ファイル構成
- 言語別ルール

---

### commit-message（コミットメッセージ）

Gitコミット時のメッセージフォーマット。

**適用タイミング**: コミット作成時

**主な内容**:
- Conventional Commits形式
- プレフィックス（feat, fix, docs等）
- 本文・フッターの書き方

---

### code-review-criteria（コードレビュー基準）

コードレビュー時のチェックポイント。

**適用タイミング**: レビュー実施時

**主な内容**:
- 機能要件の確認
- コード品質チェック
- セキュリティ（OWASP Top 10）
- パフォーマンス観点

---

### test-conventions（テスト規約）

テストコード作成時のルール。

**適用タイミング**: テストコード生成時

**主な内容**:
- テストファイル配置
- 命名規則
- AAAパターン（Arrange-Act-Assert）
- カバレッジ基準

---

### documentation-style（ドキュメントスタイル）

ドキュメント作成時のスタイルガイド。

**適用タイミング**: MDファイル作成・編集時

**主な内容**:
- Markdown記法
- 見出し構成
- コードブロック
- テンプレート

---

### doc-management（ドキュメント管理）

ドキュメントの配置・管理ルール。

**適用タイミング**: ドキュメント操作時

**主な内容**:
- ファイル配置規則
- 単一ソース原則
- 更新手順
- 同期ルール

## スキルファイル

```
.claude/skills/
├── coding-standards.md
├── commit-message.md
├── code-review-criteria.md
├── test-conventions.md
├── documentation-style.md
└── doc-management.md
```

## スキルのカスタマイズ

プロジェクト固有のルールがある場合は、スキルファイルを編集してください。

### 例：TypeScriptプロジェクト向けカスタマイズ

`coding-standards.md` に以下を追加：

```markdown
## TypeScript固有ルール

- 厳格な型定義を使用（any禁止）
- interface優先（type aliasは複合型のみ）
- enum禁止（const object使用）
```

## 新規スキルの追加

新しいスキルを追加する場合：

1. `.claude/skills/` に新規MDファイルを作成
2. このカタログに追記
3. 関連エージェントの定義を更新（必要に応じて）

### テンプレート

```markdown
# スキル名

## 概要
このスキルの目的を説明

## ルール

### カテゴリ1
- ルール1
- ルール2

### カテゴリ2
- ルール3
- ルール4

## 例

良い例と悪い例を記載
```
