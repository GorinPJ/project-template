対話形式で新規プロジェクトを作成します。

## 概要

以下の質問に順番に回答することで、プロジェクトの設定を決定し、必要なファイルを自動生成します。
各質問で「0」を選択するとその項目をスキップできます。

## 質問フロー

### 質問1: プロジェクト名
プロジェクト名を入力してもらう（英数字、ハイフン推奨）

### 質問2: プロジェクト種別
1. Webアプリケーション（フロント＋バックエンド）
2. APIサーバーのみ
3. バッチ処理/ツール
4. ライブラリ/SDK
5. PoC/プロトタイプ
6. その他

### 質問3: アーキテクチャ
1. SPA + API分離（React/Vue + バックエンドAPI）
2. SSR/フルスタックJS（Next.js / Nuxt）
3. 従来型Webアプリ（PHP / Java / Ruby / Python）
4. 静的サイト + API
5. APIのみ（バックエンド専用）
6. その他

### 質問4: 言語/フレームワーク
アーキテクチャに応じて適切な選択肢を提示

### 質問5: データベース
1. PostgreSQL
2. MySQL
3. MongoDB
4. SQLite
5. その他
0. スキップ

### 質問6: 開発環境構築
1. Docker（推奨）
2. ローカル環境
3. 両方のドキュメントを用意
0. スキップ

### 質問7: CI/CD設定
1. GitHub Actions
2. GitLab CI
3. その他
0. スキップ

### 質問8: セキュリティ設定
1. 基本設定（.env.example, .gitignore）
2. 詳細設定（+ Dependabot, セキュリティチェックリスト）
0. スキップ

### 質問9: Confluence連携
1. スペース作成 + 標準フォルダ構成
2. 標準フォルダ構成のみ（手動作成）
0. スキップ

### 質問10: 確認
設定内容を一覧表示し、以下を確認：
- yes: この内容で作成
- no: キャンセル
- edit: 項目を修正

## 生成ファイルの確認

ファイル生成前に、生成予定のファイル一覧を表示して確認を求める。

## 設定の記録

決定した内容は `.claude/project-config.yml` に記録する。
スキップした項目は `skipped` として記録し、後から個別コマンドで設定可能。

## 個別設定コマンド

スキップした項目は以下のコマンドで後から設定可能：
- `/setup-docker` - Docker環境構築
- `/setup-cicd` - CI/CD設定
- `/setup-security` - セキュリティ設定
- `/setup-confluence` - Confluence連携

## 関連ドキュメント

詳細は `.claude/docs/guides/new-project-setup-guide.md` を参照。
