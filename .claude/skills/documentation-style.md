# ドキュメントスタイル

ドキュメント作成時は以下のスタイルに従うこと。

## 基本原則

1. **簡潔に**: 必要な情報を最小限の文字数で
2. **明確に**: 曖昧な表現を避ける
3. **構造化**: 見出し、リスト、表を活用
4. **最新に**: 古い情報は削除または更新

## Markdownスタイル

### 見出し

```markdown
# 大見出し（ドキュメントに1つ）
## 中見出し（セクション）
### 小見出し（サブセクション）
```

### リスト

```markdown
## 順序なしリスト
- 項目1
- 項目2
  - サブ項目

## 順序リスト
1. 手順1
2. 手順2
3. 手順3
```

### コードブロック

```markdown
インラインコード: `const x = 1;`

コードブロック:
```javascript
function hello() {
  console.log('Hello');
}
```
```

### 表

```markdown
| ヘッダー1 | ヘッダー2 |
|----------|----------|
| 値1 | 値2 |
```

## READMEテンプレート

```markdown
# プロジェクト名

プロジェクトの概要（1-2文）

## 機能

- 機能1
- 機能2

## 必要条件

- Node.js >= 18
- Docker

## セットアップ

```bash
git clone ...
cd project
npm install
```

## 使い方

（基本的な使い方）

## 開発

```bash
npm run dev
```

## テスト

```bash
npm test
```

## ライセンス

MIT
```

## 仕様書テンプレート

```markdown
# 機能仕様書: {機能名}

## 概要
（機能の概要説明）

## 画面仕様

### 入力項目
| 項目名 | 型 | 必須 | 説明 |
|--------|-----|------|------|

### 出力項目
| 項目名 | 型 | 説明 |
|--------|-----|------|

## 処理仕様
1. （処理ステップ）

## バリデーション
| 項目 | ルール | エラーメッセージ |
|------|--------|----------------|

## 備考
```

## API仕様書テンプレート

```markdown
# API仕様書

## エンドポイント一覧

| メソッド | パス | 説明 |
|---------|------|------|

## 詳細

### GET /api/users

ユーザー一覧を取得

**リクエスト**

| パラメータ | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| limit | number | - | 取得件数 |

**レスポンス**

```json
{
  "data": [
    { "id": 1, "name": "User1" }
  ]
}
```

**エラー**

| コード | 説明 |
|--------|------|
| 401 | 認証エラー |
```

## 文章スタイル

### 敬体・常体
- 技術文書: 常体（〜である、〜する）
- ユーザー向け: 敬体（〜です、〜します）

### 避けるべき表現
- 「など」の多用
- 曖昧な形容詞（「適切な」「良い」）
- 二重否定
